## Learner Authentication APIs

- `POST /portal/validate-signup-data`

### Active Learner Flow

- Validate signup data: `POST /portal/validate-signup-data`
- Create learner account: `POST /portal/submit-learner-registration`
- Login with email/password: `POST /portal/authenticate`

### Sample Learner Login Request

```json
  "password": "YourPassword123"
}
```

## CRUD Pattern

Training hub CRUD is done through the shared `/api/*` endpoints using the Django model name in the payload.

### Create

```http
POST /api/post-data/
```

```json
{
  "model": "Course",
  "body": {
    "name": "Python Basics",
    "description": "Intro course"
  }
}
```

### Read List

```http
GET /api/get-data/?model=Course
```

### Read Single Record

```http
GET /api/get-data/?model=Course&name=Python%20Basics
```

### Update

```http
PATCH /api/patch-data/
```

```json
{
  "model": "Course",
  "body": {
    "name": "Python Basics",
    "description": "Updated description"
  }
}
```

### Delete

```http
DELETE /api/delete-doc/
```

```json
{
  "model": "Course",
  "body": {
    "name": "Python Basics"
  }
}
```

### Workflow or Docstatus Operations

- Submit: `PUT /api/submit-doc/`
- Cancel: `PUT /api/cancel-doc/`

## Notes For Teammates 

- Learner portal auth uses `/portal/*`, not `/api/*`.
- Generic system CRUD uses `/api/*` with the `model` field.
- Training hub URLs under `/app/zetrc_training_hub/*` are page routes, not direct CRUD APIs.
- For training hub data operations, use the shared `/api/get-data/`, `/api/post-data/`, `/api/patch-data/`, and `/api/delete-doc/` endpoints with the correct model name.
